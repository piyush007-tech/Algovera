Arrays Revision Notes

What is an Array?

An array is a linear data structure that stores elements of the same data type in contiguous memory locations.

[10, 20, 30, 40, 50]

Time Complexities

Access O(1)
Traversal O(n)
Search O(n)
Insertion O(n)

Must Know Patterns

  • Sliding Window
  • Prefix Sum
  • Two Pointer
  • Binary Search
  • Kadane Algorithm

Important Formulas

prefix[i] = prefix[i-1] + arr[i]
mid = low + (high-low)/2
n(n+1)/2

Delete Beginning O(1)
Search O(n)

  • Reverse List

Most Important Concepts

  • Reverse Linked List
  • Dummy Node

Dynamic Programming Revision Notes

What is Dynamic Programming?

Dynamic Programming (DP) is an optimization technique used to solve overlapping subproblems efficiently.

Solve Once, Reuse Many Times

When to Use DP?

  • Overlapping Subproblems
  • Optimal Substructure
  • Recursive Repetition
  • Optimization Problems

DP Approaches

Memoization → Top Down
Tabulation → Bottom Up
Space Optimization

Fibonacci Pattern

dp[i] = dp[i-1] + dp[i-2]

Used in: Climbing Stairs, House Robber, Fibonacci Problems.

Most Important DP Patterns

  • Fibonacci Pattern
  • Knapsack Pattern
  • LCS Pattern
  • Grid DP
  • Bitmask DP

Important DP Algorithms

0/1 Knapsack
Longest Common Subsequence
Longest Increasing Subsequence
Matrix Chain Multiplication

Grid DP Problems

  • Unique Paths
  • Minimum Path Sum
  • Cherry Pickup
  • Dungeon Game

Knapsack Formula

dp[i][w] = max(take, notTake)

Used for: Subset Sum, Coin Change, Partition Problems.

Important Complexities

Fibonacci DP O(n)
Knapsack O(n*W)
LCS O(n*m)
TSP O(2ⁿ*n)

Most Important Interview Problems

  • Climbing Stairs
  • House Robber
  • Coin Change
  • Edit Distance
  • Burst Balloons

DP Optimization Tips

  • Identify States
  • Find Transition
  • Define Base Cases
  • Optimize Space
  • Avoid Recalculation

Advanced DP Topics

  • Digit DP
  • Tree DP
  • Bitmask DP
  • DP + Binary Search
  • DP on Graphs

Hashing Revision Notes

What is Hashing?

Hashing is a technique used to store, search and retrieve data efficiently using a hash function.

Key → Hash Function → Index

Why Hashing?

  • Fast Searching
  • Efficient Lookup
  • Duplicate Detection
  • Frequency Counting

Important Components

Key
Hash Function
Hash Table
Bucket

Common Hash Function

h(key)=key % size

Converts key into hash table index.

Collision Handling

  • Chaining
  • Linear Probing
  • Quadratic Probing
  • Double Hashing

Important Complexities

Insert O(1)
Search O(1)
Delete O(1)
Worst Case O(n)

Most Important Concepts

  • Frequency Count
  • Prefix Sum + Hashing
  • Sliding Window + Hashing
  • String Hashing
  • Collision Resolution

Important Hashing Structures

HashMap → key:value
HashSet → unique keys
Unordered Map → O(1)

String Hashing

hash(s)=Σ(s[i]*p^i) mod m

Used in: Rabin Karp, substring matching, rolling hash.

Most Important Interview Problems

  • Two Sum
  • Valid Anagram
  • Group Anagrams
  • Longest Consecutive Sequence
  • Subarray Sum Equals K

Important Patterns

  • Frequency Count
  • Prefix Sum
  • HashSet Lookup
  • Sliding Window
  • String Hashing

Optimization Tips

  • Use HashMap for Fast Lookup
  • Use HashSet for Uniqueness
  • Use Prefix Sum for Subarrays
  • Handle Collisions Properly

Heap Revision Notes

What is a Heap?

A heap is a complete binary tree that follows the heap property.

Complete Binary Tree

Types of Heap

  • Min Heap
  • Max Heap
  • Binary Heap
  • Fibonacci Heap

Heap Properties

Parent ≥ Children (Max Heap)
Parent ≤ Children (Min Heap)
Complete Binary Tree

Heap Index Formulas

Left Child = 2i+1
Right Child = 2i+2
Parent = (i-1)/2

Important Complexities

Insert O(log n)
Delete Root O(log n)
Peek O(1)
Build Heap O(n)

Heap Operations

  • Insert
  • Delete Root
  • Peek
  • Heapify Up
  • Heapify Down

Priority Queue

Most common heap application where highest/lowest priority is processed first.

Implemented using Heap

Most Important Patterns

  • Kth Largest

  • Merge K Lists
  • Heap Sort
